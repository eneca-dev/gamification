# Cohere Reranker для RAG-чатбота

## Цель

Улучшить точность поиска по базе знаний: сейчас короткие или нетипично сформулированные вопросы ("что нужно делать тут?", "зачем мне нужна геймификация?") не проходят порог косинусного сходства 0.35 и бот отвечает "нет информации". Решение — расширить первичную выборку pgvector + добавить Cohere reranker как второй фильтр.

**Паттерн:** retrieve many → rerank → filter

## Затрагиваемые файлы

Только Python VPS (`d:\1work\gamification-vps-scripts\chat_agent\`):
- `requirements.txt`
- `config.py`
- `rag.py`

Изменений в Next.js, БД и схеме — нет.

## Этапы реализации

### Этап 1: Зависимости и конфиг

**Файлы:** `requirements.txt`, `config.py`

- `requirements.txt` — добавить `cohere>=5.0.0`
- `config.py`:
  - Читать `COHERE_API_KEY` из `.env`
  - Инициализировать `cohere_client = cohere.ClientV2(api_key=COHERE_API_KEY)`
  - Добавить `RERANK_MODEL = "rerank-multilingual-v3.0"`
  - Добавить `RERANK_THRESHOLD = 0.3` — минимальный `relevance_score` для включения чанка в контекст
  - Понизить `RAG_SIMILARITY_THRESHOLD` с `0.35` до `0.20` — расширяем первичную выборку
  - Увеличить `RAG_MATCH_COUNT` с `15` до `25` — больше кандидатов для реранкера

### Этап 2: Интеграция реранкера в `rag.py`

**Файл:** `rag.py`

Текущий поток `get_context()`:
```
вопрос → эмбеддинг → pgvector (порог 0.35) → контекст
```

Новый поток:
```
вопрос → синонимы → эмбеддинг → pgvector (порог 0.20, 25 кандидатов)
       → Cohere rerank (исходный вопрос без синонимов) → фильтр (score >= 0.3) → контекст
```

Детали:
- В `rerank()` передаём **исходный** вопрос (до `_expand_synonyms`), а не нормализованный — реранкер сам понимает семантику
- Документы для реранкера: `[chunk["content"] for chunk in chunks]`
- Из результата берём `result.results` — список с `index` и `relevance_score`
- Финальный список: только чанки с `relevance_score >= RERANK_THRESHOLD`, отсортированные по score
