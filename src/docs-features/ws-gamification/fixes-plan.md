**Что сделать:**
- Добавить раздел «Щит стрика» в `business-logic.md` с описанием:
  - При красном дне стрик не сбрасывается мгновенно — ставится pending reset с grace period 24 часа

**Что сделать:**
- Добавить в step 4d аналогичную проверку для L2: загрузить `budget_pending` с `status = 'approved'` и `ws_task_l2_id IS NOT NULL`
- Для каждой: проверить actual_hours (прямые + дочерние L3) vs max_time
- Если превышен → событие `budget_revoked_l2`, статус → `revoked`

**Текущее поведение:** скрипт проверяет наличие записи в `ws_daily_reports` за вчера. Если sync подтянул отчёт до запуска compute-gamification — нарушения нет.

**Что проверить:**

**Что сделать:**
- Уточнить в `business-logic.md`: у сотрудника есть время до момента синхронизации (утро следующего дня) чтобы заполнить отчёт за вчера
