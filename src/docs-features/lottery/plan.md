# Механика лотереи

## Цель

Ежемесячный розыгрыш 1 дорогого лота (мышь, кресло, клавиатура). Сотрудники покупают билеты за 💎 (неограниченно), победитель определяется рандомом. Основная цель — массовый вывод 💎 из оборота при фиксированных затратах компании (1 единица товара в месяц).

## Бизнес-правила

- Максимум 1 активная лотерея одновременно (UNIQUE constraint на month)
- Период: автоматически с 1-го числа по конец месяца
- Цена билета: задаётся при создании (по умолчанию 300), неизменяема после старта
- Количество билетов на человека: неограничено
- Процент шанса выигрыша: показывается только после покупки хотя бы 1 билета
- Возврат 💎 проигравшим: НЕТ (весь смысл — вывод из оборота)
- Отмена лотереи: невозможна после создания
- Розыгрыш: автоматически в конце месяца (Edge Function по крону)
- Победители: отображаются списком навсегда для всех участников

## Архитектура

### Новая таблица: `lottery_draws`

| Поле           | Тип                           | Описание                                                 |
| -------------- | ----------------------------- | -------------------------------------------------------- |
| id             | uuid PK                       | gen_random_uuid()                                        |
| name           | text NOT NULL                 | Название приза ("Logitech MX Master 3S")                 |
| description    | text                          | Описание приза                                           |
| image_url      | text                          | Картинка приза                                           |
| ticket_price   | integer NOT NULL, default 300 | Цена одного билета                                       |
| product_id     | uuid FK → shop_products       | Связанный товар-билет (создаётся автоматически)          |
| status         | text NOT NULL                 | 'active' / 'completed', CHECK constraint                 |
| month          | date NOT NULL, UNIQUE         | 1-е число месяца (2026-04-01). Гарантирует 1 лотерею/мес |
| winner_user_id | uuid FK → ws_users            | Победитель (NULL до розыгрыша)                           |
| drawn_at       | timestamptz                   | Когда провели розыгрыш                                   |
| created_by     | uuid FK → ws_users            | Админ-создатель                                          |
| created_at     | timestamptz default now()     |                                                          |

### Связь с существующими таблицами

- Билет лотереи = обычный `shop_product` в категории "Розыгрыши" (slug: `draw`, id: `e493b896-...`)
- Покупка билета = стандартный `purchase_product` RPC → `shop_order`
- Каждый `shop_order` для этого product = 1 билет = 1 шанс
- Подсчёт билетов: `COUNT(*) FROM shop_orders WHERE product_id = lottery.product_id AND status != 'cancelled'`
- Шанс: `мои_билеты / всего_билетов * 100%`

### Новая RPC: `draw_lottery_winner(p_lottery_id)`

1. Проверить что лотерея active
2. Выбрать рандомный shop_order: `WHERE product_id = ... AND status != 'cancelled' ORDER BY random() LIMIT 1`
3. Записать winner_user_id, drawn_at
4. Установить status = 'completed'
5. Деактивировать product (is_active = false)

## Этапы реализации

### Этап 1: БД + серверный модуль

- Миграция: таблица `lottery_draws`, RLS, RPC `draw_lottery_winner`
- Модуль lottery: types.ts, queries.ts, actions.ts, index.ts, index.client.ts
- ~7 файлов

### Этап 2: Админка лотереи

- Страница `/admin/lottery` + компонент LotteryAdmin
- Форма создания (название, описание, картинка, цена билета)
- Текущая лотерея: статистика (билетов продано, участников)
- Таблица истории + победители
- Навигация в AdminNav
- ~4 файла

### Этап 3: UI магазина — баннер, покупка билетов, победители

- LotteryBanner — приз, обратный отсчёт до конца месяца, кнопка покупки
- После покупки: "Ваши билеты: N, Шанс: X%" (через существующий PurchaseButton или расширение)
- LotteryWinners — список прошлых розыгрышей
- Интеграция в StoreClient, скрыть лотерейные товары из основной сетки
- ~5 файлов

### Этап 4: Edge Function — автоматический розыгрыш

- Крон на 1-е число каждого месяца
- Находит active лотерею за прошлый месяц → вызывает draw_lottery_winner
- Обновление документации: `src/docs/lottery.md`, `src/docs/shop.md`, `src/docs/admin.md`
- ~2 файла

## Критерии готовности

- [ ] Админ может создать лотерею с названием приза, описанием, картинкой и ценой билета
- [ ] Максимум 1 активная лотерея одновременно (constraint в БД)
- [ ] Сотрудник видит секцию лотереи в магазине
- [ ] Сотрудник может купить неограниченное количество билетов
- [ ] После покупки показывается количество билетов и % шанса
- [ ] Процент не показывается тем, кто не купил билет
- [ ] Розыгрыш происходит автоматически в конце месяца
- [ ] Победитель записывается в таблицу
- [ ] Список прошлых победителей отображается в магазине
- [ ] 💎 за билеты не возвращаются
- [ ] `npm run build` проходит без ошибок
