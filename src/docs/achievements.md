# Достижения — проектирование

## Исходные достижения (первоначальный вариант)

### 1. Worksection (Дисциплина ведения задач)

- **Эталонная дисциплина** (Индивидуальное) — 0 красных дней за 12 месяцев. Приз: настольный органайзер из архитектурного бетона. Бонус: 1000 ПК.
- **Эффективное управление** (Командное) — команда с min % красных дней за квартал. Приз тимлиду: ручка Parker с гравировкой. Бонус: 1500 ПК.
- **Образцовый отдел** (Отдельское) — отдел с min % красных дней за квартал. Приз НО: кожаный бювар с тиснением. Бонус: 2000 ПК.

### 2. Revit (Техническая эффективность)

- **Лидер автоматизации** (Индивидуальное) — топ-1 по запускам плагинов за квартал. Приз: 3D-печатная статуэтка. Бонус: 2000 ПК.
- **Технологичная команда** (Командное) — топ-1 команда по среднему кол-ву запусков на сотрудника. Приз: переходящий тотем/кубок. Бонус: +500 ПК каждому.
- **Цифровой авангард** (Отдельское) — топ-1 отдел по % вовлечённости. Приз: переходящий кубок. Бонус: пицца на отдел.

### 3. Корпоративная культура (Благодарности)

- **Поддержка коллег** — топ-1 по полученным благодарностям за квартал. Приз: стеклянная стела «Сердце команды». Бонус: 1000 ПК.
- **Межфункциональное взаимодействие** — благодарности от 3+ отделов за квартал. Приз: Power Bank софт-тач. Бонус: 600 ПК.
- **Наставничество** — 5+ благодарностей в категории «Обучение/Менторство». Приз: кожаный ежедневник. Бонус: 1200 ПК.

---

## Проблемы исходного варианта

1. **Фиксированный квартал для всех** — если человек "провалился" в начале квартала, ему нечего ждать до следующего. Теряет мотивацию.
2. **Ручной approve** — админ должен каждый раз подтверждать. Лишняя нагрузка.
3. **Топ-1 соревнования** — зависят от роли (сметчик не запускает Revit), размера отдела, количества людей.
4. **Нет прогресс-бара** — пользователь не видит, как идёт к достижению.

## Вариант 1: "Прогрессия" (3 уровня)

**Принцип:** каждый раздел = одно достижение с 3 ступенями (бронза-серебро-золото). Индивидуальный темп, автоматическое начисление, прогресс-бар. Соревнование живёт отдельно на dashboard (Leaderboard, DepartmentContest).

### WS Дисциплина — "Дисциплина"

| Уровень | Условие                            | Награда               | Сброс                                                        |
| ------- | ---------------------------------- | --------------------- | ------------------------------------------------------------ |
| Бронза  | 30 рабочих дней без красных подряд | 500 ПК                | При красном -> откат до начала текущего уровня               |
| Серебро | 90 дней без красных подряд         | 1000 ПК + приз        | При красном -> остаётся бронза, счётчик серебра сбрасывается |
| Золото  | 250 дней (год) без красных подряд  | 3000 ПК + крутой приз | При красном -> остаётся серебро                              |

Ключевое: полученный уровень нельзя потерять. Сбрасывается только прогресс к СЛЕДУЮЩЕМУ уровню.

### Revit — "Автоматизация"

| Уровень | Условие                        | Награда               | Сброс                             |
| ------- | ------------------------------ | --------------------- | --------------------------------- |
| Бронза  | 30 дней использования плагинов | 500 ПК                | Нет сброса — дни копятся суммарно |
| Серебро | 90 дней                        | 1000 ПК + приз        |                                   |
| Золото  | 250 дней                       | 3000 ПК + крутой приз |                                   |

Без сброса — каждый день использования +1. Не подряд, а суммарно.

### Благодарности — "Признание"

| Уровень | Условие                     | Награда               | Сброс      |
| ------- | --------------------------- | --------------------- | ---------- |
| Бронза  | 5 полученных благодарностей | 300 ПК                | Нет сброса |
| Серебро | 15 благодарностей           | 600 ПК + приз         |            |
| Золото  | 30 благодарностей           | 1200 ПК + крутой приз |            |

### Оценка Варианта 1

| Параметр           | Оценка | Комментарий                                       |
| ------------------ | ------ | ------------------------------------------------- |
| Логичность         | 5/5    | 3 раздела x 3 уровня, запоминается моментально    |
| Соревновательность | 2/5    | Нет соревнования, только личная гонка             |
| Мотивация          | 5/5    | Прогресс-бар, промежуточные награды, мягкий сброс |
| Реализуемость      | 5/5    | Данные есть, считается автоматически              |
| Справедливость     | 5/5    | Каждый в своём темпе                              |
| Итого              | 22/25  | Минус: нет командного духа и соревнования         |

## Вариант 2: "Бейджи + Переходящие кубки"

**Принцип:** три слоя мотивации, каждый уникальный:

| Слой                  | Что                               | Период     | Сброс             |
| --------------------- | --------------------------------- | ---------- | ----------------- |
| Стрики (уже работает) | Дни подряд -> бонус за серию      | Ежедневно  | При нарушении     |
| Бейджи (новое)        | Суммарный опыт -> бейдж           | Бессрочно  | Никогда           |
| Кубки (новое)         | Лучший отдел -> переходящий кубок | Ежемесячно | Переходит другому |

### Почему не дублирует стрики

- Стрик = "дни ПОДРЯД" (серия). Поощряет стабильность.
- Бейдж = "суммарные 💎" (накопление). Поощряет объём.
- Человек с 10 стриками по 7 дней получил 250 ПК от стриков. Общий WS-заработок = 500 💎 -> бронзовый бейдж. Разные вещи, одни данные.

Три физических кубка, переходят между отделами каждый месяц:

| Кубок               | Метрика                                            | Бонус                          |
| ------------------- | -------------------------------------------------- | ------------------------------ |
| Кубок дисциплины    | Отдел с лучшим % зелёных дней за месяц             | +10 ПК каждому                 |

- На UI: история кубков — "Янв: BIM, Фев: КР пром, Мар: ТМ"
- Кубок автоматизации уже фактически работает (fn_award_department_contest + DepartmentContest на dashboard)
- Физический кубок стоит в кабинете отдела-победителя

## Лучшие практики (анализ корпоративных систем)

### Microsoft Viva

- Badges за личные milestone-ы (не соревнования)
- Streaks с milestone-бонусами на 7, 30, 90 дней
- Рейтинги отдельно от достижений

### SAP SuccessFactors

- Achievements = пороговые (получи X, сделай Y)
- Leaderboards = отдельная механика
- Kudos с уровнями: Bronze (5), Silver (15), Gold (50)

### Duolingo

- Streaks с "freeze" (можно пропустить 1 день без потери)
- Лиги (соревнование) ОТДЕЛЬНО от достижений
- Достижения = milestone-ы (10 дней, 100 уроков)

### Habitica

- При провале: НЕ полный сброс, а потеря части прогресса
- Гильдии (команды) — общий прогресс-бар к общей цели

### Salesforce Trailhead

- Badges (порог): пройди X -> получи badge
- Ranks: накопительные уровни
- Нет "топ-1" в достижениях

### 5 ключевых паттернов

1. **Достижения != Рейтинги** — везде разделено
2. **Прогрессия уровней** — ступени, не одна далёкая цель
3. **Мягкий сброс** — не полная потеря, а откат на уровень ниже
4. **Streak freeze** — возможность сохранить серию при пропуске
5. **Командные достижения = общая цель**, не соревнование между командами
