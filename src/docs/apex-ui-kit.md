# Apex UI Kit — Справочник дизайн-системы

Документация по Figma UI kit стороннего проекта Apex. Используется как **источник вдохновения для оформления** нашего проекта геймификации.

> **Важно:** структура нашего проекта (сайдбар, карточки, дашборд, лейаут страниц) **остаётся прежней**. Apex-стиль применяется только к визуальному оформлению элементов — форма кнопок, цвета, инпуты, теги, состояния компонентов.

Оригинальные изображения: `C:\Users\Саша\Downloads\ui-kit-extracted\` (24 PNG-файла).

## Типографика

- **Шрифт:** системный sans-serif (в Figma — Inter или аналог). В нашем проекте — Nunito (совместимо).
- **Сглаживание:** antialiased

| Роль | Размер | Вес | Применение |
|---|---|---|---|
| Заголовок секции | 20px / 1.25rem | 700 | Заголовки на страницах |
| Подзаголовок | 16px / 1rem | 600 | Названия карточек |
| Основной текст | 14px / 0.875rem | 400 | Контент, описания |
| Мелкий текст | 12px / 0.75rem | 400–500 | Лейблы, подписи, метаданные |
| Кнопки | 14px / 0.875rem | 500–600 | Текст кнопок |

## Компоненты

### Button (Кнопки)

Файл в Figma: `Buttons.png`

**Форма:** все кнопки — `rounded-full` (капсула).

#### Варианты

| Вариант | Фон | Текст | Граница | Применение |
|---|---|---|---|---|
| **Primary** | `--apex-primary` (#1B6B58) | белый | нет | Основное действие: «Сохранить», «Создать» |
| **Secondary** | белый | `--apex-text` | `--apex-border` | Второстепенное действие: «Отмена», «Нет» |
| **Stroke** | прозрачный | `--apex-primary` | `--apex-primary` | Добавление: «+ Добавить позицию», «Раздел из шаблона» |
| **Ghost** | прозрачный | `--apex-text-secondary` | нет | Навигационные: «Узнать больше ›» |
| **Danger** | прозрачный | `--apex-danger` | нет | Деструктивные: «Удалить разделов: 4» (иконка + текст) |

#### Состояния (для всех вариантов)
- **Default** — базовое состояние
- **Hover** — Primary темнее (`--apex-primary-hover`), Secondary получает teal-обводку
- **Pressed** — немного темнее hover
- **Disabled** — серый фон, серый текст, `cursor-not-allowed`
- **Focused** — синее кольцо `ring-2 ring-[--apex-focus] ring-offset-2`

#### Размеры
- **md (default):** `px-6 py-2.5 text-sm`
- **sm:** `px-4 py-1.5 text-xs`

#### С иконками
- Иконка слева или справа от текста, `gap-2` между иконкой и текстом
- Danger-кнопка: иконка корзины слева, красный текст

## Скриншоты Apex-приложения (контекст)

> Скриншоты чужого приложения — нужны **только для понимания**, как Apex-стиль выглядит в живом продукте. Копировать лейаут и структуру страниц не нужно.

### Страница входа (для вдохновения)

- Белый фон, split: текст слева, иллюстрация справа
- Кнопка входа: `rounded-full`, Primary стиль — **это можно перенести в наш проект**
- Авторизация через OAuth, без формы

### Хедер и навигация (для вдохновения)

- Активный пункт меню — пилюля с границей цвета primary — **интересный паттерн для нашего сайдбара**
- Иконки справа: уведомления, аватар, выход

## Как реализовать компоненты

Когда понадобится перенести Apex-стиль в проект:

1. **Добавить CSS-токены** — скопировать все `--apex-*` переменные из таблицы выше в блок `:root` в `globals.css`
2. **Создать** `src/components/apex/` с компонентами (Button, Input, SearchInput, Tag, Tabs, Card, Checkbox, Notification)
3. **Создать** `src/app/ui-kit/page.tsx` — страницу-витрину на `/ui-kit`
4. **Шрифт:** текущий Nunito совместим, менять не обязательно
5. **Иконки:** Lucide React уже в проекте, все иконки из набора там есть
6. **Подсказка:** сказать Claude «реализуй компоненты по документу `src/docs/apex-ui-kit.md`» — достаточно для полного воспроизведения системы
